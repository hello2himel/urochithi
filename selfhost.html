Prerequisites

Before you begin, create free accounts on these three services:

  1. GitHub — to host the source code
    • Sign up at github.com if you don't have an account
  2. Neon — for the PostgreSQL database and authentication
    • Sign up at neon.tech
    • Free tier includes: 0.5 GB storage, 60,000 monthly active users for auth
    • No credit card required
  3. Netlify — for hosting and serverless functions
    • Sign up at netlify.com (use "Sign up with GitHub" for easiest setup)
    • Free tier includes: 100 GB bandwidth, 125K serverless function requests/month
💡 No coding required! This guide is step-by-step with screenshots in mind. You'll only need to edit one small config file and paste a few values.
1

Fork the Repository

Forking creates your own copy of the Urochithi code on GitHub.

  1. Go to github.com/hello2himel/urochithi
  2. Click the "Fork" button in the top-right corner of the page
  3. On the "Create a new fork" page:
    • Leave the repository name as urochithi (or rename it to whatever you want)
    • Optionally add a description
    • Make sure "Copy the main branch only" is checked
  4. Click "Create fork"
  5. Wait a few seconds — GitHub will redirect you to your new fork
✓ Done: You now have your own copy at github.com/YOUR_USERNAME/urochithi. All future edits will be made to this copy.
Fork on GitHub →
2

Create a Neon Project

Neon provides the PostgreSQL database that stores all messages and user data.

2.1 — Create the project

  1. Go to console.neon.tech and log in
  2. Click "New Project" (the big button on the dashboard)
  3. Fill in the project details:
    • Project name: urochithi (or any name you like)
    • Postgres version: leave default
    • Region: pick the one closest to your audience (e.g., US East or Europe West)
  4. Click "Create Project"

2.2 — Copy the connection string

  1. After creation, Neon shows a dialog with your connection details
  2. Find the "Connection string" — it looks like:
    postgresql://neondb_owner:abc123xyz@ep-cool-name-12345.us-east-2.aws.neon.tech/neondb?sslmode=require
  3. Click the copy icon to copy it
  4. Paste it somewhere safe (a text file, notes app, etc.) — you'll need it in Step 5
⚠️ Keep this connection string secret! It contains your database password. Never share it publicly or commit it to GitHub. You'll paste it as an environment variable in Netlify (which keeps it private).
💡 No SQL needed: You don't need to create any tables manually. Urochithi creates them automatically the first time someone uses the app.
Open Neon Console →
3

Enable Neon Auth

Neon Auth handles user sign-up and sign-in for the dashboard. It's built into Neon — no external auth service needed.

  1. In the Neon Console, open the project you created in Step 2
  2. In the left sidebar, click the "Auth" tab
  3. Click "Enable Auth"
  4. Wait a moment — Neon will set up the auth service for your project
  5. Once enabled, you'll see an Auth URL that looks like:
    https://ep-cool-name-12345.auth.neon.tech
  6. Copy this Auth URL and save it alongside your connection string from Step 2
✓ Done: Neon Auth is enabled! Unlike external auth providers, there are no callback URLs to configure, no API keys to generate, no separate dashboard to manage. Just one URL.
💡 What Neon Auth provides:
  • Email/password sign-up and sign-in
  • Session management
  • Auth data stored in the same database as your messages
  • Free tier: up to 60,000 monthly active users
4

Deploy to Netlify

Netlify hosts your site and runs the serverless functions that talk to the database.

4.1 — Log in to Netlify

  1. Go to app.netlify.com
  2. Click "Sign up" or "Log in"
  3. Choose "Sign up with GitHub" for the easiest experience
  4. Authorize Netlify to access your GitHub account when prompted

4.2 — Import your forked repository

  1. On the Netlify dashboard, click "Add new site"
  2. Click "Import an existing project"
  3. Click "Deploy with GitHub"
  4. If this is your first time, Netlify will ask to install the Netlify GitHub app:
    • Click "Configure the Netlify app on GitHub"
    • Choose "All repositories" or select just urochithi
    • Click "Install"
  5. Find and click on your urochithi repository in the list

4.3 — Configure build settings

  1. On the "Site configuration" screen, leave all settings at their defaults:
    • Branch to deploy: main
    • Build command: (leave empty)
    • Publish directory: (leave empty or .)
  2. Click "Deploy [site name]"

4.4 — Wait for deployment

  1. Netlify will start building and deploying your site (takes about 1-2 minutes)
  2. You'll see a progress indicator — wait for it to show "Published"
  3. Once done, you'll get a URL like https://random-name-123.netlify.app
  4. Note this URL — you'll need it for Step 6
⚠️ The site won't fully work yet! You still need to add the database connection and auth URL as environment variables. That's the next step.
Deploy on Netlify →
5

Set Environment Variables

Environment variables securely store your database credentials and auth URL. Netlify keeps them private — they're never exposed to visitors.

  1. In your Netlify site dashboard, click "Site configuration" in the top nav
  2. In the left sidebar, click "Environment variables"
  3. Click "Add a variable" → "Add a single variable"
  4. Add the first variable:
    • Key: DATABASE_URL
    • Value: paste the connection string from Step 2
  5. Click "Add a variable" again for the second variable:
    • Key: NEON_AUTH_URL
    • Value: paste the Auth URL from Step 3
DATABASE_URL = postgresql://neondb_owner:abc123@ep-cool-name-12345.us-east-2.aws.neon.tech/neondb?sslmode=require NEON_AUTH_URL = https://ep-cool-name-12345.auth.neon.tech
⚠️ Double-check:
  • Variable names must be exactly DATABASE_URL and NEON_AUTH_URL (all uppercase, with underscores)
  • No extra spaces before or after the values
  • The DATABASE_URL should start with postgresql://
  • The NEON_AUTH_URL should start with https://
✓ Done: Both environment variables are set. Now you need to redeploy for them to take effect (after customizing your config in the next step).
6

Customize Your Site

Edit the config file to personalize your Urochithi instance with your site name, tagline, and Neon Auth URL.

  1. Go to your forked repository on GitHub (github.com/YOUR_USERNAME/urochithi)
  2. Navigate to the file: js/config.js
    • Click the js folder
    • Click config.js
  3. Click the pencil icon ✏️ (top right of the file view) to edit
  4. Update the values:
const CONFIG = { // Your default username (shows on the homepage postage stamp) username: "your_username", // Your site's display name siteName: "YOUR SITE NAME", // Tagline below the site name siteTagline: "Your custom tagline here", // Maximum characters per message (2000 is a good default) maxMessageLength: 2000, // Leave these as-is onboardingUrl: "/onboard.html", // Your Netlify site URL (from Step 4) siteUrl: "https://your-site-name.netlify.app", // Neon Auth URL (from Step 3) neonAuthUrl: "https://ep-cool-name-12345.auth.neon.tech" };
  1. Scroll down and click "Commit changes..."
  2. In the popup, click "Commit changes" to save
💡 What each field does:
  • username — your default letterbox username
  • siteName — displayed in the header of every page
  • siteTagline — subtitle shown on the letter form page
  • siteUrl — your Netlify URL (used for generating share links)
  • neonAuthUrl — the Auth URL from Neon (used by the login form)
7

Redeploy

After setting environment variables and editing the config, you need to trigger a fresh deploy so everything takes effect.

Option A: Automatic (if you edited config.js on GitHub)

If you committed changes to config.js on GitHub in Step 6, Netlify will automatically redeploy within a minute. You can check the deploy status on your Netlify dashboard.

Option B: Manual trigger

If the automatic deploy didn't trigger, or you only changed environment variables:

  1. Go to your site on the Netlify dashboard
  2. Click the "Deploys" tab
  3. Click "Trigger deploy" → "Deploy site"
  4. Wait for the deployment to finish (status shows "Published")
✓ Done: Your site is now fully configured and live! The environment variables are loaded and the config is applied.
8

Test Everything

Let's verify everything is working correctly.

8.1 — Check the homepage

  1. Visit your site URL (e.g., https://your-site.netlify.app)
  2. You should see the homepage with your site name, intro text, and action buttons

8.2 — Create your account

  1. Click "Open Your Letterbox" (or go to /dashboard.html)
  2. Click "Sign Up"
  3. Enter your name, email, and a password (min 8 characters)
  4. Click "Sign Up"
  5. You'll be prompted to choose a username
  6. Pick something like yourname — this becomes your letterbox URL
  7. Click "Claim Username"
✓ Account created! You should now see your empty dashboard with the message "No letters yet."

8.3 — Send a test letter

  1. Open a new tab (or incognito window)
  2. Go to https://your-site.netlify.app/yourname (replace yourname with the username you chose)
  3. You should see the letter form with "to: @yourname" at the top
  4. Type a test message and click "Send Letter"
  5. You should see: "Your letter has been sent successfully!"

8.4 — Check the dashboard

  1. Go back to your dashboard tab
  2. Click "↻ Refresh"
  3. Your test message should appear!

8.5 — Share your letterbox

  1. On the dashboard, find the "📎 Your letterbox URL" bar
  2. Click "📋 Copy" to copy your unique URL
  3. Share it anywhere — social media bio, group chats, email signature, etc.
🎉 Congratulations! Your self-hosted Urochithi instance is fully operational! Anyone who visits your letterbox URL can send you anonymous letters.
9

Custom Domain (Optional)

By default, your site is at random-name.netlify.app. You can change this or add your own domain.

9.1 — Change the Netlify subdomain

  1. In the Netlify dashboard, go to "Site configuration"
  2. Under "Site details", click "Change site name"
  3. Enter your preferred name (e.g., my-letters → my-letters.netlify.app)
  4. Click "Save"

9.2 — Use your own domain

  1. Go to "Domain management" in Netlify
  2. Click "Add a domain"
  3. Enter your domain (e.g., letters.yourdomain.com)
  4. Follow Netlify's instructions to add the DNS records at your domain registrar
  5. Netlify will automatically provision a free SSL certificate
💡 Remember: After changing your domain, update siteUrl in js/config.js to match the new URL, then commit and redeploy.

Migrating from v1 (Google Sheets)

If you were using the previous version of Urochithi with Google Sheets and time-based PINs, follow these steps to migrate your data.

Export from Google Sheets

  1. Open your Google Sheet that contains the messages
  2. Go to File → Download → Comma-separated values (.csv)
  3. Save the .csv file to your computer

Import into Urochithi v2

  1. Log into your new dashboard at /dashboard.html
  2. Click the 📥 Import button in the header
  3. Click "Choose File" and select the CSV you downloaded
  4. Click "Import"
  5. You'll see a status message showing how many messages were imported
💡 CSV Format: The importer automatically detects columns named "Timestamp", "Message", and "Session ID" (case-insensitive). As long as your CSV has at least a "Message" column, the import will work. Original timestamps are preserved.

Export your data anytime

Click the 📤 Export button in the dashboard header to download all your messages as a CSV file at any time.

Clean up old environment variables

These v1 variables are no longer needed and can be safely removed from Netlify:

  • GSCRIPT_URL
  • DASHBOARD_PIN
  • TIME_PIN_ALGORITHM
  • RECAPTCHA_SECRET_KEY
✓ No data lost: All your messages from Google Sheets are now safely stored in Neon PostgreSQL.

Troubleshooting

"Internal Server Error" when sending a letter

  • Check that DATABASE_URL is set correctly in Netlify environment variables
  • Make sure the Neon project is active (not paused due to inactivity on free tier)
  • Verify you redeployed after adding environment variables

"Sign in failed" on the dashboard

  • Verify NEON_AUTH_URL is set correctly in Netlify environment variables
  • Confirm neonAuthUrl in js/config.js matches the Auth URL in your Neon Console
  • Check that Neon Auth is enabled in your Neon project (Auth tab in the console)

Letters sent but not appearing in dashboard

  • Make sure the username in the URL (e.g., /yourname) matches the username you chose during registration
  • Click "↻ Refresh" on the dashboard
  • Check that the sender visited your-site.netlify.app/yourname (not just the homepage)

Site shows the homepage instead of the letter form

  • The homepage shows at the root URL (/)
  • The letter form shows at /username — make sure you're visiting the URL with a username in the path

Netlify deploy failed

  • Check the deploy logs in Netlify for specific errors
  • Make sure you didn't accidentally break the JSON in package.json
  • Try triggering a fresh deploy: Deploys → Trigger deploy → Clear cache and deploy site

Neon database paused / not responding

  • Free-tier Neon projects auto-pause after 5 minutes of inactivity
  • They automatically wake up on the next request — the first request may take 1-2 seconds longer
  • This is normal behavior and doesn't affect user experience
