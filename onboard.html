Prerequisites

Before you begin, make sure you have:

  • A GitHub account (free)
  • A Neon account for PostgreSQL database and authentication (free tier: 60,000 MAU) — neon.tech
  • A Netlify account (free, can sign up with GitHub) — netlify.com
💡 Note: No coding experience required! Just follow the steps carefully.
1

Fork the Repository

Start by creating your own copy of the Urochithi project on GitHub.

  1. Visit the Urochithi GitHub Repository
  2. Click the "Fork" button in the top-right corner
  3. Select your account as the destination
  4. Wait for GitHub to create your fork (takes ~10 seconds)
✓ Success: You now have your own copy at github.com/YOUR_USERNAME/urochithi
Fork on GitHub →
2

Set Up Neon Database

Create a free PostgreSQL database on Neon to store messages and user data.

  1. Go to Neon Console and sign up / log in
  2. Click "New Project"
  3. Name it something like "urochithi"
  4. Select a region close to you
  5. Click "Create Project"
  6. Copy the Connection String — it looks like:
    postgresql://user:pass@ep-xxx.region.aws.neon.tech/neondb?sslmode=require
⚠️ CRITICAL: Save the connection string somewhere safe. You'll need it in Step 6!
💡 Note: The database tables are created automatically when the first message is submitted. No manual SQL needed!
Open Neon Console →
3

Enable Neon Auth

Enable Neon Auth to handle user authentication for the dashboard. Neon Auth is built on Better Auth and stores auth data in your Neon database — no external service needed.

  1. Go to Neon Console
  2. Open the project you created in Step 2
  3. Navigate to the Auth tab in the left sidebar
  4. Click Enable Auth
  5. Copy the Auth URL — it looks like:
    https://your-project-id.auth.neon.tech
💡 That's it! Unlike external auth providers, Neon Auth is built right into your Neon project. No callback URLs, no API identifiers, no separate dashboard — just one URL. Auth data lives in the same database as your app data. Free tier supports up to 60,000 monthly active users.
✓ Done: Neon Auth is enabled! You have your Auth URL.
Open Neon Console →
4

Customize Your Site

Edit the configuration file to personalize your Urochithi instance.

  1. Go to your forked repository on GitHub
  2. Navigate to js/config.js
  3. Click the pencil icon ✏️ to edit
  4. Update these values:
const CONFIG = { username: "your_username", // Default recipient username siteName: "YOUR SITE NAME", // Name of your letter site siteTagline: "Your custom tagline", // Tagline below name maxMessageLength: 2000, // Max characters (keep as-is) onboardingUrl: "/onboard.html", // Keep as-is siteUrl: "https://your-site.netlify.app", // Your deployed URL // Neon Auth settings (from Step 3) neonAuthUrl: "https://your-project-id.auth.neon.tech" // Neon Auth URL };
  1. Scroll down and click Commit changes
✓ Done: Your site is now personalized with your information!
5

Deploy to Netlify

Connect your GitHub repository to Netlify for automatic deployment.

5.1 Create Netlify Account (if needed)

  1. Go to Netlify
  2. Click Sign up and choose Sign up with GitHub
  3. Authorize Netlify to access your GitHub account

5.2 Import Your Repository

  1. On the Netlify dashboard, click Add new site → Import an existing project
  2. Click Deploy with GitHub
  3. Find and select your urochithi repository
  4. Leave all build settings as default (empty)
  5. Click Deploy [your-site-name]

Netlify will start deploying your site. This takes 1-2 minutes.

💡 Important: After deploying, note your Netlify site URL. You'll need to update siteUrl in js/config.js with your actual Netlify URL.
✓ Success: Your site will be live at https://random-name-123.netlify.app
You can customize this URL later in Netlify settings.
Deploy on Netlify →
6

Configure Environment Variables

Set the server-side environment variables in Netlify to connect your database and Neon Auth.

  1. In your Netlify site dashboard, go to Site configuration
  2. Click Environment variables in the left sidebar
  3. Add the following variables:
DATABASE_URL = postgresql://user:pass@ep-xxx.region.aws.neon.tech/neondb?sslmode=require NEON_AUTH_URL = https://your-project-id.auth.neon.tech
⚠️ Important: After adding the environment variables, you MUST redeploy:
  1. Go to Deploys tab
  2. Click Trigger deploy → Deploy site
  3. Wait for the new deployment to finish (~1 minute)
💡 Troubleshooting: If you get errors later, double-check:
  • Variable names are exactly as shown (case-sensitive)
  • DATABASE_URL is the complete Neon connection string
  • NEON_AUTH_URL matches the Auth URL from the Neon Console Auth tab
  • You redeployed after adding the variables
7

Test & Launch

Time to test your anonymous letter platform!

7.1 Register Your Account

  1. Visit your-site.netlify.app/dashboard.html
  2. Click "Sign In"
  3. Sign up with your email and password (or log in if you already have an account)
  4. Choose your username — this becomes your letterbox URL
  5. Your letterbox URL will be: your-site.netlify.app/your-username

7.2 Send a Test Message

  1. Visit your letterbox URL (e.g., your-site.netlify.app/your-username)
  2. Type a test message in the textarea
  3. Click Send Letter
  4. You should see: "Your letter has been sent successfully!"

7.3 Check Your Dashboard

  1. Go to your-site.netlify.app/dashboard.html
  2. Sign in (if not already)
  3. You should see your test message!
🎉 Congratulations! Your anonymous letter platform is live and working!

7.4 Share Your Letterbox

Share your letterbox URL from the dashboard. Add it to your social media bio, or wherever you want to receive anonymous messages!

💡 Multi-User: Other people can also sign up! They'll get their own letterbox at your-site.netlify.app/their-username. Each user only sees their own messages.

Migrating from v1 (Google Sheets)

If you were using the previous version with Google Sheets, you can easily import your existing data.

Export from Google Sheets

  1. Open your Google Sheet with the messages
  2. Go to File → Download → Comma-separated values (.csv)
  3. Save the file to your computer

Import into Urochithi v2

  1. Log into your dashboard at /dashboard.html
  2. Click the 📥 Import button in the header
  3. Select your CSV file
  4. Click Import
  5. Your messages will be imported with their original timestamps!
💡 CSV Format: The importer automatically detects columns named "Timestamp", "Message", and "Session ID" (case-insensitive). As long as your CSV has at least a "Message" column, the import will work.

Export from v2

You can also export your data at any time from the dashboard using the 📤 Export button. This downloads all your messages as a CSV file.

✓ No Data Lost: The old environment variables (GSCRIPT_URL, DASHBOARD_PIN, TIME_PIN_ALGORITHM, RECAPTCHA_SECRET_KEY) are no longer needed and can be safely removed from Netlify.
